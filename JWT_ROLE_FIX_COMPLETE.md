# JWT Role Claim Fix - Complete

## Problem
The `/api/submissions/{packageId}/submit` endpoint was returning **403 Forbidden** error when called by Agency users, even though they were authenticated.

## Root Cause
The JWT token generated by `AuthService` contained the role as a **numeric enum value** (`"role":"0"`) instead of the **role name string** (`"role":"Agency"`).

```csharp
// BEFORE (WRONG):
var token = GenerateToken(user.Id, user.Email, user.Role.ToString());
// This generated: "role":"0" for Agency, "role":"1" for ASM, "role":"2" for HQ
```

The endpoint authorization was checking for the role name:
```csharp
[Authorize(Roles = "Agency")]
```

This caused a mismatch - the token had `"role":"0"` but the endpoint expected `"role":"Agency"`.

## Solution
Updated `AuthService.cs` to use `Enum.GetName()` to get the enum name instead of its numeric value:

```csharp
// AFTER (CORRECT):
var roleName = Enum.GetName(typeof(Domain.Enums.UserRole), user.Role) ?? user.Role.ToString();
var token = GenerateToken(user.Id, user.Email, roleName);
// This generates: "role":"Agency", "role":"ASM", or "role":"HQ"
```
